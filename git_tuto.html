What is Git and GitHub?

  • Git is a distributed version-control system for tracking changes in source code during software development.
  • GitHub is a web-based hosting service for version control using Git.

Refesh on thr basis Linux commands

  • pwd to check your repository
  • ls to list your repository files
  • cd to change the parent directory
  • cd < directory > to change to a directory
  • touch < file > to creat a file
  • mkdir < directory > to make a directory
  • cat < file > to open a file

Basis Git commands

  • git config --global user.name "Votre nom ou pseudo"
  • git config --global user.email "Votre@email.com"
    • to create an account and an email which will be also use for GitHub.
  • git add < file > to add a file
  • git add . to add all files in the repository
  • git status to get the status of your Git
  • git commit -m “message” to register modifications in Git
  • git commit -a -m “message” with -a you update an already existing file in Git (Add all the files already in the index and commit them)
  • git log to see the history of all commit
  • git checkout < SHADuCommit > to position yourself on a given commit of your history
  • git checkout master to go back to the main branch (this is the most recent commit)
  • git revert < SHADuCommit > to create a new commit that is the opposite of the previous
  • git commit --amend -m “message” to modify the message of the last commit
  • git reset --hard to cancel the change made if the commit has not already commited
  • git clone < HTTPS-link-Github > to add/download a project via a HTTPS-link from GitHub to the local machine
  • git push origin master to send changes to your GitHub repository
  • git pull origin master to retrieve locally(from Github to your computer) the last changes in the repository GitHub
  • git branch to see all your branches
  • git branch [ name_of_new_branch ] to create a new branch
  • git checkout [ branch_name ] to position yourself in branch (branch-name) in your repo.
  • git checkout -b [ branch_name ] to group the two previous commands in one command
  • git merge < another_branch > to add to the current branch the changes you made in another branch (another_branch). This will put modifications in another in current
  • git mergetool vimdiff tool to solve merge conflits. Conflits can also be resolve manually. commit after resoltion of conflits. No need of commit message, git provide message for conflits
  • git blame < file > it's a more direct way to find out who made a particular change in a file
  • git show < SHA > to directly obtain the details of the desired commit by entering the beginning of its sha (sha got with git blame). Ex.: git show 05b1233
  • git stash to save the current changes (version not save) without any commit, in order to be able to return to the latest version to commit or even to change branch.
  • git stash pop to return to the branch/file where you were before doing 'git stash', in order to finish and commit your changes. Be careful, pop empties the stash
  • git stash apply if you want to keep the changes in your stash, you can use apply instead of pop
  • vim < file > can help you to edit your file
  • touch .gitignore creates a gitignore file. This file contains the (names) files to ignore. To ignore a file (the configuration file for example) by Git or adding it to the .gitignore file.
  • fork on Githup to copy a project to his own Githup repository

Contributing: Example

  • 1. Fork it
  • 2. Create your feature branch (git checkout -b my-new-feature)
  • 3. Commit your changes (git commit -am 'Add some feature')
  • 4. Push to the branch (git push origin my-new-feature)
  • 5. Create new Pull Request: that is to propose a modification to a project hosted on GitHub (per message using create pull request on GitHub)
